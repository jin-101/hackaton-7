# 데이터 연동 추가 요구 사항 프롬프트

전체적으로 mock data를 사용하고 있는 부분을 전면 수정해야 해.
현재 구성되어 있는 DB 데이터를 기반으로 현재 mock data를 사용하고 있는 부분을 DB 데이터를 가져와서 표출되도록 수정해줘.
DB 데이터가 제대로 구성되어 있지 않은 부분은 현재 실제 대한항공 운임 등 비슷하게 더미 데이터를 추가해줘
그리고 만약 페이지에서 공급석이나 항공권 운임 가격을 수정한다면 수정한 가격이 DB에 반영되도록 해야해.

페이지 확인해보니까 전체적으로 데이터가 없어서 표출되지 않아. 데이터 없는 부분은 더미 데이터를 DB에 추가해서 페이지에서 정상적으로 데이터가 표출되도록 수정해줘. 그리고 다양한 필터링 적용했을 때 데이터가 변동되는게 그래프로 보였음 좋겠어.

대시보드에서 '등급별 평균 LF (국내선 전체)' -> 'Load Factor 범례 (국내선 전체)'로 항목이 변경되었는데 이전처럼 '등급별 평균 LF (국내선 전체)'가 표출되도록 수정해주고, 일자별 필터링 수정시 항공편별 LoadFactor 관련 데이터가 변동되도록 데이터도 수정해줘

지금 api 호출시 다 502 Bad Gateway가 발생하는데 정상적으로 호출되도록 ai-dlc방법론 적용해서 오류 수정해줘

# v7 추가 요구사항

## 전체 (라우팅)

- 새로고침했을 때 선택한 탭 유지되도록 페이지 라우팅해줘
- `http://localhost:5173/#/report` 처럼 `/#/` 경로는 제거하고 바로 `/fares`, `/report`, `/dashboard` 이런식으로 URL이 표출되도록 수정해줘

## 대시보드

- DB 데이터를 연동하면서 대시보드에서 등급별 평균 LF 차트가 단순 색상 범례로 변경되어 있는데, 이전처럼 '등급별 평균 LF (국내선 전체)' 항목으로 등급별 평균 LF 데이터를 바 차트로 표출되도록 복원해줘
- 일자별 필터링 (1일/3일/7일/10일) 변경 시 항공편별 Load Factor 데이터가 해당 기간 기준으로 변동되도록 수정해줘 (기존에는 항상 당일 기준 고정이었음)


- 운임 관리 탭에서 특정 항공편 선택해서 상세 페이지 진입한 후 새로고침시에도 상세 페이지에 머물도록 해당 페이지도 라우터 적용을 해줘
- 새로고침 클릭시 고객이 좌석 변경, 환불, 구매하는 행위에 따라 데이터가 변경되도록 눈에 보이도록 구성해줘 그리고 겉보이게 실제 대한항공 홈페이지 내 데이터와 유사하게 구성해줘
- 경쟁사 모니터링 탭에서 'DB 연동' 칩은 제거해줘

## 운임 관리 (버그픽스)

- 운임 관리 탭에서 항공편명이 UUID(예: `a1b2c3d4`) 형태의 이상한 값으로 표시되는 문제 수정
  - **원인**: DB 연동 후 `DashboardFlight.id`가 DB의 `flight_id`(8자리 UUID 축약)로 바뀌었는데, 편명 표시 위치에서 `f.id`를 그대로 사용하고 있었음
  - **수정**: `DashboardFlight` 인터페이스에 `flightNo: string` 및 `route: string` 필드 추가. 편명 표시 위치(운항현황 목록, 상세 헤더, Profit Analysis 패널, 좌석 등급별 운임 관리 제목) 전체를 `f.flightNo` / `selectedFlight.flightNo`로 교체. `buildDashboardFlights()` mock 반환 객체에도 `flightNo`, `route` 필드 추가.
