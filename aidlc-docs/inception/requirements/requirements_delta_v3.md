# [jin v2] 좌석 수 관리 로직 최신화 (requirements_delta_jin.md 2차 병합)

## 운임관리 — Open / Closed / Sold Out 로직 개편
1. Closed 처리 시 좌석 자동 이관 로직 제거. 상태만 전환, 좌석 수 수정은 수동으로 열어둠.
2. Sold Out 상태에서도 좌석 수 수정 가능 (운임 수정도 허용).
3. 총 좌석 수 불변 원칙: 어떠한 경우에도 전체 좌석 합계는 초기 설정값 유지.
4. 프레스티지 석: 어떤 경우에도 좌석 수 수정 불가 (잠금).
5. 일반석 좌석 수 변경 시 AI 재배분 로직:
   - 특정 등급 좌석 증가 → AI가 이익 극대화 기준으로 다른 일반석 중 기회비용 최소 등급(가격×잔여석 최소)에서 차감
   - 특정 등급 좌석 감소 → AI가 이익 극대화 기준으로 다른 일반석 중 수익 기여 최대 등급(가격×잔여석 최대)으로 이관
   - 각 등급의 좌석 수는 점유된 좌석 수보다 작을 수 없음. 같아지는 경우 Sold Out으로 자동 전환
   - 차감/이관할 여유가 없는 경우 alert 배너 표시 ("감소 가능한 좌석이 부족합니다" 등)
