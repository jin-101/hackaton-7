# Requirements Delta v8 — 인벤토리 통제 로직 수정 및 날짜별 상태 격리

**날짜**: 2026-05-22  
**기준 버전**: requirements.md (v7 Phase7까지 통합)

## 변경 항목

### 1. FR-01: Sold Out 처리 수정 (기존 내용 오류 수정)

**기존 (틀림)**:
> `**Sold Out 처리**: 운임·좌석 수 수정 모두 허용. 상태 토글만 불가`

**수정**:
> `**Sold Out 처리**: 운임 수정 불가(잠금). 좌석 수 수정 가능(증가 허용). 상태 토글 불가`

---

### 2. FR-01: 좌석 총합 초과 시 에러 표출 (신규)

- 특정 등급의 좌석 수를 늘릴 때, 변경 후 전체 등급 좌석 수의 합이 해당 항공편 전체 좌석 수(aircraft capacity)를 초과하는 경우 변경을 차단하고 에러 배너를 표시해야 함
- 에러 메시지 형식: `"좌석 변경 불가: 전체 좌석 수(N석)를 초과합니다. 다른 등급의 좌석을 먼저 줄여주세요."`
- 이 검증은 AI 재배분(`aiReallocateSeats`) 수행 전에 사전 차단하는 방식으로 적용됨
- 기존 `**총 좌석 수 불변 원칙**`을 강제하는 사용자 가시적 에러 처리

---

### 3. FR-01: 인벤토리 확정 버튼 저장 운임 명세 추가

- **인벤토리 실시간 통제 확정 버튼** 클릭 시:
  - AI 추천 운임(`aiPrice`)이 현재 운임(`price`)과 다른 클래스: **AI 추천 운임으로 저장**
  - AI 추천 운임이 없거나 동일한 클래스: 현재 운임 그대로 저장
  - 전 클래스에 대해 순차적으로 `PUT /fares/{flightId}` API 호출
  - 저장 성공 후 로컬 상태도 `price = aiPrice`로 동기화 (화면 즉시 반영)
- 확정 버튼은 AI 추천 유무와 관계없이 항상 표시 (좌석 수 조정 후 확정하는 경우 포함)

---

### 4. FR-01: 날짜별 AI 적용 완료 상태 격리 (신규)

- "적용 완료" 배지 및 AI 추천 상세 보기 버튼 숨김 상태는 **날짜별로 독립적으로 관리**되어야 함
- 특정 날짜(예: 5/22)에서 AI 추천을 모두 적용해 "적용 완료" 상태가 된 경우:
  - 다른 날짜(예: 5/25)로 이동 시 해당 날짜의 "적용 완료" 상태가 보이지 않아야 함
  - 다시 5/22일로 돌아왔을 때 "적용 완료" 상태가 **유지**되어야 함 (날짜 복귀 시 리셋 아님)
- 새로고침 버튼 클릭 시에만 전체 상태 초기화
