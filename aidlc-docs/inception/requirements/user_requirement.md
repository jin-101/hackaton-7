# 프로그램 기능 요구사항(System Requirements)
Pricing 담당자가 실제로 사용할 시스템에 포함되어야 할 기능입니다.

# # 1 실시간 대시보드(Real-time Dashboard)
- 노선별 실시간 예약 현황 및 AI 권장 가격 표시
- 경쟁사 가격 변동에 따른 긴급 알림(Alert) 기능

# # 2 시뮬레이션 모듈(What-if Analysis)
- 특정 변수(예: 유가 10% 상승, 신규 경쟁사 진입) 변경 시 수익 변화를 예측하는 시뮬레이션 환경

# # 3 자동 가격 승인 및 수동 개입(Hybrid Control)
- AI가 산정한 가격이 일정 범위를 벗어나지 않으면 자동 확정
- 비정상 상황(천재지변 등) 발생 시 담당자가 즉시 수동으로 가격 범위를 제한할 수 있는 기능

# # 4 보고서 자동 생성(Automated Reporting)
- 수익 증대 기여도(Yield Management Report), 판매 목표 대비 달성률 자동 분석

# 추가 지시사항 및 페르소나 프롬프트
난 항공사 RM팀 담당자야
국내선 전 노선에 대하여 AI를 통한 실시간 가격책정 변화를 통해 의미있는 Revenue를 확보하는게 나의 일이야
일을 편하게 하기위해 프로그램을 만들어야 하는데 네 도움이 필요해.
우선 노선은 대한항공이 운항하는 국내선 노선이 대상이고
운항스케쥴에 따라 일자별로 가격을 통제하는게 기본 방식이야.
아침시간, 낮시간, 저녁이후 시간의 가격은 과거 통계를 근거로 하여 기본가격이 책정되고
월별로는 성수기 시즌과 비성수기 시즌 등으로 나뉘어 가격이 산정돼
그리고 좌석의 등급은 프레스티지를 시작으로 일반석 정상가, 일반석 할인가, 일반석 특가로 구분되어지고
항공기는 모두 B737-900으로 운항할 예정이야.. 737-900은 프레스티지 8석에 이코노미가 165석으로 구성되어 있어.
일자별 기본요금을 위 기준이 반영되도록 구성해줘

이렇게 기본가격을 기준으로 해서 현재 상황을 AI로 분석하여 가장 적절한 가격책정을 해야만 해
판매율에 대한 실시간 캡쳐를 통해 L/F를 계산하여 대응해야 하고, 경쟁사 판매현황을 가지고 대응해야 하고,
국제뉴스나 국내뉴스 등 항공수요에 영향이 있을만한 동향을 실시간 분석하여 대응해야해.
유가는 특히 중요하고 
이벤트, 축제, 운동경기 등 수요와 관련된 사항도 대응해야해.

한국어로 표시해주고 
달력을 통해 표시되는 일주일간의 날짜를 변경할 수 있게 해줘. L/F는 일자의 각 운항편에 종속되게 해주고
여기에 담당자가 AI에게 분석을 요청하는 공간을 만들어줘. 요청은 불특정한 요인에 대해서 문의할 것이고 
그 질의에 대하여 AI는 가격변동의 요인이 있는지 없는지를 판단하고 가격결정이 필요하면 어떻게 변경하겠다는 
질의를 사용자에게 전달한 후 사용자가 최종 결정할 수 있게 해줘

현재 대시보드에서 해당편의 Cost와 revenue를 나타나게 해줘 (L/F와 관련 있겠지?)

그리고 왜 이런 결정을 분석을 하게 되었는지를 꼭 알려줘야 해. 그래야 최종 가격을 담당자가 결정할 수 있겠지.

여기에 담당자가 AI에게 분석을 요청하는 공간을 만들어줘. 
요청은 불특정한 요인에 대해서 문의할 것이고 그 질의에 대하여 AI는 가격변동의 요인이 있는지 없는지를 판단하고 
가격결정이 필요하면 어떻게 변경하겠다는 질의를 사용자에게 전달한 후 사용자가 최종 결정할 수 있게 해줘

이런 점을 고려해 담당자가 쉽게 인지하고 가격책정을 할 수 있는 시스템을 만들어야 해.
