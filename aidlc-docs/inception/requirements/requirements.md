## 1. 의도 분석 요약 (Intent Analysis)

- **사용자 요청**: 항공사 Revenue Management를 위한 가격관리 프로그램
- **요청 유형**: New Project (Greenfield)
- **범위 추정**: Multiple Components (가격 관리, AI/ML 추천, 대시보드, Inventory 관리, 시뮬레이션, 보고서)
- **복잡도 추정**: Complex (AI/ML 가격 추천 + 실시간 대시보드 + 시뮬레이션 + 자동 승인 하이브리드)
- **프로젝트 목적**: 해커톤/데모/프로토타입 — 핵심 기능 우선, 빠른 구현

## 2. 이해관계자 및 사용자

### 주요 사용자
- **Revenue Manager (Pricing 담당자)**: 항공사 내부 수익 관리 담당자 (= 관리자 겸 사용자)

  - AI 추천 가격 검토 및 승인/거부
  - 수익 현황 모니터링
  - 비정상 상황 시 수동 가격 범위 제한

## 3. 기능 요구사항 (Functional Requirements)

### FR-01: 운임 관리 (Fare Management)
- 국내선 항공권 운임 관리 (김포-제주, 김포-부산, 김포-여수, 인천-제주 등)
- 가격 산정 단위 구분:
  - 프레스티지(비즈니스), 일반석 정상, 일반석 할인, 일반석 특가
  - 선호 시간대 반영 요금 구분
  - 주중/주말 요금 구분
  - 성수기/비성수기 구분
- 좌석 서브클래스(Booking Class) 관리: Y, B, M, K, H, Q 등 세부 등급별 운임 설정
- 항공편(Flight) 단위 운임 조회, 등록, 수정
- Leg(구간) 기반 Inventory 관리 (좌석 가용 수량)
- 항공운임 기본 가격 설정 (전년도 데이터 활용 또는 AI 임의 설정)
- **책정된 항공운임 직접 가격 조정**: 내부 사정으로 AI 책정 운임을 수동 수정하거나, AI 운임 책정 전 특정 노선/날짜 가격 고정 기능
- 클래스별 운임 조회 (그래프 형태 표출)

### FR-02: AI/ML 기반 가격 추천 및 하이브리드 자동 승인 (Hybrid AI Price Control)
- ML 모델을 활용한 최적 가격 자동 계산
- **하이브리드 자동 승인**:
  - AI 산정 가격이 설정된 허용 범위 내이면 자동 확정
  - 허용 범위 초과 시 Revenue Manager에게 추천 가격 제시 (수동 승인 필요)
- 추천 근거 설명 제공 (탑승률 예측, 수요 트렌드, 경쟁사 가격, 예약 리드타임 등)
- 담당자가 추천 가격 승인 시 시스템 반영, 거부 시 현행 유지
- **비정상 상황 대응 (천재지변 등)**: 담당자가 즉시 수동으로 가격 상·하한 범위를 제한할 수 있는 운영 안전장치 (어드민 기능)

### FR-03: 경쟁사 가격 모니터링 (Competitor Price Monitoring)
- AI가 생성한 Mock 경쟁사 가격 데이터 활용
- 특정 노선을 운항하는 국내 모든 항공사 조회
- 동일 시즌, 동일 시간대 기준 경쟁사 가격 현황 조회
- 자사 vs 경쟁사 가격 비교 뷰
- 외부 요인은 웹 정보 기반으로 최대한 실시간성 확보

### FR-04: 실시간 대시보드 (Real-Time Dashboard) ✅ v2 구현
> **UI 기준**: `basic_guide.html` — 대한항공 RM AI 커맨드 센터 레이아웃

#### 화면 레이아웃 (3컬럼 그리드)
- **좌측 사이드바** (col-span-3):
  - 노선 선택 드롭다운 (대한항공 국내선 전 노선: GMP-CJU, GMP-PUS, GMP-CJJ, GMP-TAE 등 9개 노선)
  - 주간 달력 피커: 7일 날짜 버튼 (이전/다음 주 이동 가능), 성수기 표시, 오늘 강조
  - **AI 전략 분석 요청 패널** (대한항공 네이비 #002561 배경): 텍스트 입력창 + 분석 시작 버튼
  - 최근 8일 일별 수익 소형 바 차트
- **중앙 메인** (col-span-6):
  - **AI 긴급 전략 제안 팝업** (분석 완료 시 표출): 분석 근거 텍스트, 승인/기각 버튼
  - **운항 현황 테이블**: 편명/시간/시간대(아침·오전·오후·저녁), L/F 바, Pace(전주 대비%), 현재가, AI추천가, 상태(수요급증/안정적/수요저조/위험)
  - **좌석 등급별 현황 카드 그리드** (2×2): 프레스티지(C8석)/일반석 정상가(Y·B·M)/일반석 할인가(S·K·L)/일반석 특가(T·V·G) — 각 카드에 Open/Closed/Sold Out, 현재가, AI추천가, 판매율 바, 판매석/전체석
  - **AI 분석 근거** 텍스트 (선택 편 기준)
- **우측 사이드바** (col-span-3): Profit Analysis 패널
  - REVENUE (현재 판매 기준)
  - COST (운항 고정비)
  - PROFIT / MARGIN % 바 게이지
  - 인벤토리 실시간 통제 확정 버튼
- **고정 하단 푸터**: Daily Est. Revenue, Avg Margin, 선택 편 요약, 전 노선 인벤토리 동기화 버튼

#### 기체 및 좌석 기준
- **기종**: B737-900 고정 (프레스티지 8석 / 이코노미 165석 = 총 173석)
- **좌석 등급 4단계**: 프레스티지(C) → 일반석 정상가(Y/B/M) → 일반석 할인가(S/K/L) → 일반석 특가(T/V/G)

#### 시간대별 기본가 기준
- 아침(~09:00): 수요 선호 시간대, 기본가 대비 +20~25%
- 오전(09:00~12:00): 비선호 시간대, 기본가 기준
- 오후(12:00~17:00): 보통 수요, 기본가 기준
- 저녁(17:00~): 수요 선호 시간대, 기본가 대비 +30~35%

#### AI 분석 요청 워크플로우
1. 담당자가 돌발 이슈 자유 입력 (태풍, 행사, 연휴 등)
2. AI가 가격 변동 요인 판단 → 변경 필요 시 전략 제안 팝업 표출
3. 담당자가 제안 내용 확인 후 **승인(전략 적용)** 또는 **기각** 선택
4. 승인 시 해당 편 클래스 운임·인벤토리 즉시 반영

#### KPI 및 데이터
- **L/F**: 판매석/전체석 × 100 (%), 편별로 색상 구분(90%+ 적색, 70%+ 황색, 그 외 네이비)
- **Pace**: 전주 동일편 대비 예약 증감률 (%, +/- 표시)
- **Cost**: 항공기임차료 + CREW비용 + 공항사용료 + 기타 고정비 (노선별 차등)
- 새로고침 주기: 수동 동기화 버튼 제공 (토큰 비용 고려)
- BI 도구 연동 제외 (프로토타입 범위)

### FR-05: 시뮬레이션 모듈 (What-if Analysis)
- 특정 변수 변경 시 수익 변화 예측:
  - 유가 상승/하락 (%)
  - 신규 경쟁사 진입
  - 가격 인상/인하 시 예상 수요 및 수익 변화 분석 → 최적 수익 구간 도출
  - 주변 변수 대응: 경쟁사, 유가, 환율 등 외부 변수 실시간 분석 및 대응 가격 제안
- **시뮬레이션 고정값**: 기종, 항공기임차료, CREW비용, 공항사용료 등 Cost 항목은 고정값으로 처리
- 과거 데이터 및 실제 예약 시스템 연계 정보는 AI 가상치로 대체

### FR-06: 가격 이력 관리 (Price History)

- 변경 일시, 변경자, 변경 전/후 가격 기록

### FR-07: 알림 기능 (Notifications/Alerts)

- 판매 목표 대비 달성률 자동 분석

## 7. 주요 비즈니스 규칙 (Business Rules)

- BR-01: 상위 Booking Class 운임은 하위 클래스보다 항상 높아야 한다
- BR-02: Load Factor = 예약 좌석 수 / 전체 좌석 수 × 100 (%)
- BR-03: AI 추천 가격은 현행 가격의 -30% ~ +50% 범위를 초과할 수 없다 (수동 승인 기준 범위)
- BR-04: 운임은 0보다 커야 한다 (음수 불가)
- BR-05: 승인된 가격 변경은 즉시 시스템에 반영된다
- BR-06: AI 자동 확정 가격은 사전 설정된 허용 범위(상·하한) 내에서만 자동 적용된다
- BR-07: 비정상 상황 발생 시 담당자가 설정한 수동 가격 범위가 AI 추천보다 우선한다
- BR-08: 프레스티지 > 일반석 정상 > 일반석 할인 > 일반석 특가 순으로 운임이 책정된다
- BR-09: 주말 요금은 주중 요금보다 높을 수 있으며, 성수기 요금은 비성수기보다 높다

## 8. UI/UX 구조

- **단일 페이지 구조** (Single Page Application)
- **탭 기반 네비게이션**:
  - 탭 1: 대시보드 (Dashboard) — 실시간 현황, KPI, 차트

## 10. 제약사항 (Constraints)

- 해커톤 데모용: 핵심 기능 우선, 완전한 운영 환경 불필요
- 동시 사용자 10명 이하 기준 설계
- Mock 데이터 기반 AI/ML (실제 항공사 데이터 연동 불필요)
- AI/ML 모델 일부는 더미 데이터로 대체 가능 (토큰 사용량 고려)
- 새로고침 주기는 기본 하루 단위 (토큰/비용 최적화)
- 외부 요인 데이터는 웹 정보 기반 Mock으로 대체

## 11. 논의 필요 사항 (Open Questions)

- **월별 vs 일별 운임 표출 단위**: 기본 월별 표출 + 드릴다운으로 일단위 조회 권장
