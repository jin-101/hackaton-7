- **세부현황(Step 2) 기존 기능 유지**: 좌석 등급별 운임 관리(ClassEditCard) 기능은 step2에서 그대로 제공

- **헤더 항공편명·출발시간 제거**: 새로고침 버튼 우측에 표출되던 항공편명과 출발 시간 UI 요소 제거

#### 기체 및 좌석 기준

- **기종**: B737-900 고정 (프레스티지 8석 / 일반석 정상 30석 / 일반석 할인 85석 / 일반석 특가 50석 = 총 173석)
- **좌석 등급 4단계**: 프레스티지(C, 8석) → 일반석 정상(Y, 30석) → 일반석 할인(M, 85석) → 일반석 특가(V, 50석)
- **총 좌석 수 불변 원칙**: 어떠한 경우에도 전체 좌석 합계는 항공편 초기 설정값과 동일해야 함
- **프레스티지 좌석 수 잠금**: 어떤 상태(Open/Closed/Sold Out)에서도 좌석 수 수정 불가
- **Closed 처리**: 상태만 전환, 좌석 자동 이관 없음. 운임 수정 불가(잠금), 좌석 수는 수동 수정 가능
- **Sold Out 처리**: 운임 수정 불가(잠금). 좌석 수 수정 가능(증가 허용). 상태 토글 불가
- **일반석 좌석 수 AI 재배분**:
  - 증가 시: 기회비용 최소(가격×잔여석 최소) 일반석 등급에서 차감
  - 감소 시: 수익 기여 최대(가격×잔여석 최대) 일반석 등급으로 이관
  - 좌석 수가 점유석과 같아지면 해당 등급 자동 Sold Out 전환
  - 차감/이관 불가 시 alert 배너 표시
  - **좌석 총합 초과 방지**: 특정 등급 좌석 수 증가 시 전체 등급 합이 항공편 총 좌석 수를 초과하면 변경 차단 및 에러 배너 표시 (`"좌석 변경 불가: 전체 좌석 수(N석)를 초과합니다. 다른 등급의 좌석을 먼저 줄여주세요."`)

### FR-01-1: RM 다이나믹 프라이싱 알고리즘 (Dynamic Pricing & Inventory Control)
- **EMSRb(Expected Marginal Seat Revenue-b) 알고리즘**: 일반석 클래스별 보호 수준(Protection Level) 및 예약 제한량(Booking Limits) 산정
  - 백엔드: scipy.stats.norm 역누적분포함수 활용 (`backend/app/core/emsr.py`)
  - 프론트엔드: A&S 26.2.17 유리 근사식 기반 `_normInv()` 내장 (`frontend/src/data/mockData.ts`)
  - 좌석 생성 시(`buildDashboardFlights`)와 수동 수정 시(`aiReallocateSeats`) 모두 EMSRb 적용
  - CV 가변: LF≥80→0.20, LF≥60→0.25, LF<60→0.40 (탑승률 기반 수요 예측 신뢰도 반영)
  - 클래스 코드 미표출: 운임 관리 카드에서 C/Y/M/V 배지 숨김, 등급명만 표시
- **Logit WTP 구매 확률 모델**: `P(Buy|p) = exp(α - β·p) / (1 + exp(α - β·p))` — 가격 민감도 β는 DTD에 반비례
- **프레스티지석 Pace-Based Continuous Pricing**: 잔여 좌석 수에 따른 가격 팩터 산정, 잔여 4석 이하 시 지수적 가격 상승
- **일반석 Hybrid Up-pricing**: 특가(V) 수요 급증 시 클래스 즉시 차단 대신 M 클래스 하한선까지 가격 상향 조정
- **통합 최적화 API**: `POST /api/rm/optimize` — DTD, 잔여 좌석, 경쟁사 가격, 검색 트래픽 비율 입력 → 클래스별 권장 운임 + 예약 한도 반환
- **시나리오 시뮬레이션 API**: `POST /api/rm/simulate-scenario` — A(비수기 저조) / B(이벤트 수요 급증) / C(경쟁사 가격 인하) 3개 시나리오 지원

### FR-02: AI/ML 기반 가격 추천 (AI Price Recommendation)

- ML 모델을 활용한 최적 가격 자동 계산
- AI 추천은 **수동 승인 전용** (자동 확정 기능 현재 미구현 — 향후 확장 가능)
- 추천 근거 설명 제공 (탑승률 예측, 수요 트렌드, 경쟁사 가격, 예약 리드타임 등)
- 담당자가 추천 가격 승인 시 시스템 반영, 거부 시 현행 유지
- AI 추천 목록은 운임 관리(FareManagement) 화면에 통합 표시 (별도 탭 없음)
- AI 추천 상세 보기: dimmed 오버레이 + 중앙 모달 형태 (현재가/추천가/분석근거/판매율)
- **거부 처리**: 거부 시 "AI 추천 거부됨" 배지만 표시. 음영(opacity) 없이 운임·좌석 수동 편집은 허용
- **날짜별 적용 완료 상태 격리**: "적용 완료" 배지 및 AI 추천 상세 보기 버튼 숨김 상태는 날짜별로 독립 관리. 다른 날짜로 이동 시 해당 날짜의 상태만 표시. 날짜 복귀 시 이전 상태 유지(리셋 안 됨). 새로고침 버튼 클릭 시에만 전체 초기화
- AI 가격 추천 외 **AI 좌석 수 추천**: 전체 공급 좌석 고정 상태에서 AI가 수익 극대화 방향으로 등급별 좌석 수 배분 추천
- **비정상 상황 대응**: 담당자가 즉시 수동으로 가격 상·하한 범위를 제한할 수 있는 운영 안전장치

### FR-03: AI 전략 분석 요청 (AI Strategy Analysis)

- 담당자가 돌발 이슈를 자유 텍스트로 입력 (태풍, 행사, 연휴, 경쟁사 동향 등)
- **'AI 전략 분석 시작' 버튼 클릭 시 AI model을 실제로 호출하여 응답 반환** — 하드코딩 응답 제거, 실제 AI model API 연동 필요
- AI가 가격 변동 요인 판단 → 변경 필요 시 전략 제안 팝업 표출 (dimmed 오버레이 + 중앙 모달)
- 제안 내용: 분석 근거 텍스트 + 가격 변동 방향 + 승인/기각 버튼
- 담당자가 **승인(전략 적용)** 또는 **기각** 선택 후 최종 반영
- 분석 대상 외부 요인: L/F(탑승률), 경쟁사 동향, 유가, 국내외 뉴스, 이벤트/축제/운동경기 등

  - AI 긴급 전략 제안 팝업 (분석 완료 시 표출)
  - 운항 현황 테이블: 편명/시간/시간대(아침·오전·오후·저녁), L/F 바, Pace(전주 대비%), AI추천가, 상태(수요급증/안정적/수요저조/매진임박) — "현재가" 컬럼 없음
  - 좌석 등급별 현황 카드 그리드 (2×2): 각 카드에 Open/Closed/Sold Out, 현재가, AI추천가, 판매율 바, 판매석/전체석
  - AI 분석 근거 텍스트 (선택 편 기준)
- **우측 사이드바** (col-span-3): Profit Analysis 패널
  - REVENUE (현재 판매 기준)
  - COST (운항 고정비)
  - PROFIT / MARGIN % 바 게이지
  - **인벤토리 실시간 통제 확정 버튼**: 클릭 시 전 클래스에 대해 순차적으로 `PUT /fares/{flightId}` API 호출. AI 추천 운임(`aiPrice`)이 있는 클래스는 aiPrice로 저장, 없으면 현재 운임 그대로 저장. 성공 후 로컬 상태도 동기화(price=aiPrice). 확정 버튼은 AI 추천 유무와 관계없이 항상 표시. 모든 항공편/클래스 조합에서 FareTier 오류 없이 정상 동작해야 함
- **고정 하단 푸터**: Daily Est. Revenue, Avg Margin, 선택 편 요약, 전 노선 인벤토리 동기화 버튼

#### KPI 및 데이터

- **L/F**: 판매석/전체석 × 100 (%), 편별 색상 구분 (90%+ 적색, 70%+ 황색, 그 외 네이비)
- **Pace**: 전주 동일편 대비 예약 증감률 (%, +/- 표시)
- **Cost**: 항공기임차료 + CREW비용 + 공항사용료 + 기타 고정비 (노선별 차등)

- **일자 필터링 데이터 연동**: 1일/3일/7일/10일 필터 변경 시 '평균 Load Factor', 'AI 승인 대기' 등 모든 KPI 데이터가 해당 기간 기준으로 변동됨
- **수익 KPI 콤마 표기**: '최근 N일 수익' KPI 카드 금액에 1,000 단위마다 콤마 삽입 (만원 단위 표기 시에도 콤마 적용)
- **DB 연동 (v7)**: `GET /dashboard/summary?route_id=X&days=N` API 호출로 실제 Flight + FareTier DB 데이터 표시. Mock 하드코딩 제거.

## 7. 주요 비즈니스 규칙 (Business Rules)

- **BR-01**: 상위 Booking Class 운임은 하위 클래스보다 항상 높아야 한다
- **BR-02**: Load Factor = 예약 좌석 수 / 전체 좌석 수 × 100 (%), 편별 종속
- **BR-03**: AI 추천 가격은 현행 가격의 ±30% 범위를 초과할 수 없다
- **BR-04**: 운임은 0보다 커야 한다 (음수 불가)
- **BR-05**: 승인된 가격 변경은 즉시 시스템에 반영된다
- **BR-06**: AI 자동 확정 가격은 사전 설정된 허용 범위(상·하한) 내에서만 자동 적용된다 (향후 구현)
- **BR-07**: 비정상 상황 발생 시 담당자가 설정한 수동 가격 범위가 AI 추천보다 우선한다
- **BR-08**: 프레스티지 > 일반석 정상 > 일반석 할인 > 일반석 특가 순으로 운임이 책정된다
- **BR-09**: 주말 요금은 주중 요금보다 높을 수 있으며, 성수기 요금은 비성수기보다 높다
- **BR-10**: Sold Out 등급은 가격 수정 불가, 좌석 수만 수정 가능
- **BR-11**: 등급별 좌석 수의 합은 항공편 전체 좌석 수를 초과할 수 없으며, 초과 시 에러로 차단

## 8. UI/UX 구조 (현재 구현 상태 기준)

- **단일 페이지 구조** (Single Page Application)
- **5탭 사이드바 네비게이션**:
  - 탭 1: 대시보드 — 실시간 현황, KPI, 차트
  - 탭 2: 운임 관리 — 운임 조회·수정, AI 추천 목록 통합
  - 탭 3: 경쟁사 모니터링 — 노선별 경쟁사 운임 현황
  - 탭 4: 시뮬레이터 — What-if 분석
  - 탭 5: 보고서 — 자동 생성 보고서 미리보기 및 다운로드
- 별도 관리자 페이지 없음 (Pricing 담당자 = 관리자 겸 사용자)
- 브랜드 컬러: 대한항공 네이비 `#002561`

## 9. 범위 외 사항 (Out of Scope)

- GDS/OTA 연동 (Amadeus, Sabre 등)
- 국제선 운임 관리
- 화물 운임 관리
- O&D(Origin & Destination) 기반 네트워크 최적화
- BI 도구 연동 (Tableau, Power BI 등)
- 다중 항공사 SaaS 기능
- 완전 자동 가격 적용 without 범위 설정

## 10. 제약사항 (Constraints)

- 해커톤 데모용: 핵심 기능 우선, 완전한 운영 환경 불필요
- 동시 사용자 10명 이하 기준 설계
- Mock 데이터 기반 AI/ML (실제 항공사 데이터 연동 불필요)
- AI/ML 모델 일부는 더미 데이터로 대체 가능 (토큰 사용량 고려)
- 새로고침 주기는 기본 하루 단위 (토큰/비용 최적화)
- 외부 요인 데이터는 웹 정보 기반 Mock으로 대체
